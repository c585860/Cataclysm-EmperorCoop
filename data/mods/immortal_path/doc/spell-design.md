## 分类

设定中，各种释放的spell实际区分为法术、秘术、神通。
- 法术基本没有学习要求
- 秘术往往要求学习特定的功法后才能学习，一般同时记录在功法的秘籍中
- 神通是功法衍生的特殊能力，不能学习而是直接获得

> **首个范例·火弹术（法术）**：`immortal_path_fireball`。定位为「法术」——凡能引气入体者皆可习练，故**不设 `spell_class`**（读残卷不会凭空把人变成修士），施放门槛由 `energy_source: qi`（没灵气放不出）天然承担。残卷比整套五行功法传承常见、价格更低。

### 等级

总的来说，术仍然可以靠练习来提升。但和大魔法等模组不一样的是，术的熟练度对效果的影响应该控制在合理范围内，主要是一定程度上节省时间和提高成功率，不会因为你的大量训练就提升。取而代之的是由你的**灵气强度**和**属性极化**作为绝对的评判标准。**灵气强度**主要受境界和装备提升，而**属性极化**很大程度和有属性偏向的功法/装备挂钩。

### 威力公式（落地）

灵气强度与属性极化都走 fork 的字符串键 enchantment 通道（`u_enchant_val`，与灵气上限同一套机制，引擎自动多来源累加），各有一个 jmath 读取函数（`jmath/qi_formulas.json`）：

- **灵气强度** `immortal_path_qi_power()` = `u_enchant_val('qi_power', 100)`，基础 100。境界载体特性「炼气士」按炼气层数 add（层数×8，炼气圆满约 196）；法宝/丹药走 add 或 multiply。法术伤害取 **强度系数 = qi_power / 100**（基础修士 1.0、炼气圆满约 2.0）。
- **属性极化** `immortal_path_polarization_<phase>()` = `u_enchant_val('polarization_<phase>', 1)`，基础 1（中性，无偏向）。八系（金木水火土冰风雷）各一条独立通道；五行功法载体特性各 add 自己那系 +0.3（→1.3）。冰/风/雷暂无功法，通道先就位、值恒为 1。

**范式**（以火弹术为例，`damage_type: heat`、火行）：
```
min_damage = round( 基础下限 × (qi_power/100) × polarization_fire )
max_damage = round( 基础上限 × (qi_power/100) × polarization_fire )
```
- 只读了残卷的普通人：系数 1.0、极化 1 → 基础威力。
- 炼气圆满 + 修火行功法：系数≈2、极化≈1.3 → 约 2.6 倍。威力由「境界×功法」驱动，与练习次数无关，正合「熟练度只省时间/提成功率」的设定。
- `max_level` 保留，spell 仍随练习升级——引擎据此降施法时间、提成功率，但**不**进伤害公式。
